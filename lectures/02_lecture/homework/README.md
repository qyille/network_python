## 🚀 Как выполнять

### Шаг 1. Прочитайте лекцию

Откройте `lecture.ipynb` — в ней объяснены HTTP, REST, Pydantic, FastAPI.

### Шаг 2. Запустите примеры

```bash
cd examples
# 01-04 — от простого FastAPI до BaseModel
python 01_initial/main.py
python 04_add_post/main.py
python 05_base_model/main.py

# demo_tests — как тестировать
python -m pytest demo_tests/test_main.py -v

# demo_gun — как сравнивать производительность
cd demo_gun && python gun.py -n 100 -c 30
```

### Шаг 3. Реализуйте задания

В каждом каталоге есть `task.py` с описанием.

**Не меняйте:**
- Названия функций и эндпоинтов (их вызывают тесты)
- Вспомогательные функции (помечены `# НЕ МЕНЯТЬ`)

**Меняйте:**
- Всё остальное — структуру кода, модели, обработчики

### Шаг 4. Проверьте локально

```bash
cd lectures/02_lecture/homework
python -m pytest -v
```

Зелёные тесты ✅ — можно пушить.

## 🧪 Что проверяют тесты

| Проверка | Описание |
|----------|----------|
| **Корректность CRUD** | Все эндпоинты работают по REST-правилам |
| **HTTP-статусы** | Правильные коды: 200, 201, 204, 400, 404, 409 |
| **Валидация** | Pydantic-модели reject'ят невалидные данные (422) |
| **Обработка ошибок** | Кастомные исключения, читаемые сообщения |
| **Производительность** | Понимание разницы async vs sync |

## 📋 Задания

### 01_bookstore — CRUD API для книжного магазина

Спроектировать и реализовать REST API для управления каталогом книг.
Подсказок нет — только спецификация эндпоинтов.

**Что нужно сделать:**
- Pydantic-модели для книг и категорий с валидацией полей
- CRUD-эндпоинты с правильными HTTP-методами и статус-кодами
- Поиск/фильтрация через query-параметры
- Кастомная обработка ошибок

### 02_errors_and_tests — Ошибки, обработка исключений и тестирование

Дано готовое FastAPI-приложение (app.py) с несколькими проблемами:
- Некорректные HTTP-статусы
- Отсутствует обработка ошибок
- Нет валидации

**Что нужно сделать:**
- Проанализировать app.py и найти все проблемы
- Исправить их в task.py
- Написать тесты, которые покрывают все эндпоинты
- Добавить один асинхронный и один синхронный эндпоинт с измерением времени
